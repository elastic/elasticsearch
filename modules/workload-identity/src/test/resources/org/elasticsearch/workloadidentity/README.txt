These PEM files are reused from modules/reindex/src/test/resources/org/elasticsearch/reindex/
and were generated by elasticsearch-certutil (see that module's README.txt). They are colocated
with HttpsWorkloadIdentityIssuerClientTests so the test can resolve them with the framework's
ESTestCase#getDataPath / getResourceDataPath helpers (which look up resources relative to the
test class's package). They provide fixed mTLS material for the test:

  ca.crt                       - test CA that signs both certs below (same content as the
                                  reindex module's ca.pem; renamed here for consistency with
                                  the .crt/.key naming used by the issuer and node material).
  issuer/issuer.crt + .key     - server cert (originally "http", password "http-password"),
                                  presented by the mock workload-identity-issuer server.
                                  SANs cover localhost and 127.0.0.1.
  node/node.crt + .key         - client cert (originally "client", password "client-password"),
                                  presented by the test client (simulating an Elasticsearch node).
                                  Subject "CN=client".

If these need to be regenerated, follow the procedure in
modules/reindex/src/test/resources/org/elasticsearch/reindex/README.txt, then rename and copy
the resulting files into this directory:

  ca.pem            -> ca.crt
  http/http.crt     -> issuer/issuer.crt
  http/http.key     -> issuer/issuer.key
  client/client.crt -> node/node.crt
  client/client.key -> node/node.key

The "http-password" and "client-password" passphrases must be preserved (the tests reference
them as String literals) or updated in lockstep with the test code.
